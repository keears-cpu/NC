# natal.kr Public Site Blueprint

이 문서는 `natal.kr` 공개 페이지를 최소 운영 가능 상태로 만들기 위한 구조안이다.

목표는 세 가지다.

1. 서비스가 실제로 운영되는 것처럼 보이게 한다.
2. PortOne 실연동 심사 전 제출 가능한 기본 페이지를 갖춘다.
3. 차트 신청, 상품 안내, 결제 후 열람 흐름이 한 번에 이해되게 한다.

## 1. 최소 페이지 맵

### A. 메인

URL 예시:

- `/`

역할:

- 서비스 소개
- 상품 요약
- 이용 흐름 안내
- 차트 신청 CTA

필수 섹션:

- 히어로
- 서비스 소개
- 보고서 유형 소개
- 이용 흐름 4단계
- 샘플 미리보기
- 자주 묻는 질문
- 푸터

### B. 상품 소개

URL 예시:

- `/products`

역할:

- 상품별 차이 설명
- 대상 독자 구분
- 가격 안내
- 결제 전 기대치 정리

필수 섹션:

- 상품 카드 4종
- 누구에게 맞는지
- 포함 항목 비교표
- 열람 방식 안내

### C. 차트 신청

URL 예시:

- `/apply`

역할:

- 출생 정보 입력
- 상품 선택
- 보고서 열람 코드(4자리) 설정
- 차트 저장
- 이후 결제 단계 진입

필수 섹션:

- 입력 폼
- 상품 선택
- 열람 코드 설명
- 신청 후 안내 문구

### D. 보고서 열람

URL 예시:

- `/report-access`

역할:

- `record_id + 보고서 열람 코드(4자리)`로 본인 보고서 열기

필수 섹션:

- record_id 입력
- 열람 코드 입력
- 결제 완료 안내
- 문의 링크

### E. 환불 / 취소 정책

URL 예시:

- `/refund`

역할:

- 디지털 상품 환불 기준 안내
- 제작 전/후 기준 설명

### F. 개인정보처리방침

URL 예시:

- `/privacy`

역할:

- 출생 정보, 연락처, 결제 정보 처리 목적 명시

### G. 이용약관

URL 예시:

- `/terms`

역할:

- 서비스 제공 범위
- 유의사항
- 책임 제한

## 2. 메인 페이지 추천 구조

### Hero

- 제목:
  - `당신의 차트를, 읽을 수 있는 보고서로`
- 부제:
  - `출생 데이터를 바탕으로 핵심 성향, 관계의 흐름, 성장 방향을 구조화된 리포트로 제공합니다.`
- CTA:
  - `차트 신청하기`
  - `상품 보기`

### 서비스 소개

- 핵심 메시지:
  - `natal.kr은 차트를 계산하는 데서 멈추지 않고, 실제로 읽고 활용할 수 있는 보고서 형태로 정리하는 서비스입니다.`

### 보고서 유형

- 기본 보고서
- 성인 심층 보고서
- 청소년 성장 보고서
- 유아·아동 기질 리포트

### 이용 흐름

1. 출생 정보를 입력합니다.
2. 원하는 보고서 상품을 선택합니다.
3. 보고서 열람 코드(4자리)를 설정합니다.
4. 결제 완료 후 내 보고서를 확인합니다.

### 샘플 안내

- 실제 보고서 일부 화면 예시
- 디자인 톤
- 어떤 식으로 읽히는지 예시

### FAQ

- 어떤 정보를 입력해야 하나요?
- 결제 후 언제 볼 수 있나요?
- 보고서는 어떻게 열람하나요?
- 환불은 가능한가요?

## 3. 상품 구조 제안

### core_report

- 이름:
  - `기본 보고서`
- 설명:
  - 핵심 성향과 방향을 빠르게 확인하는 기본형

### adult_deep_blueprint

- 이름:
  - `성인 심층 보고서`
- 설명:
  - 자기이해, 내면 패턴, 관계와 방향성까지 더 깊게 읽는 심층형

### teen_growth_report

- 이름:
  - `청소년 성장 보고서`
- 설명:
  - 학습 방식, 진로 방향, 정서, 부모 가이드를 함께 보는 성장형

### child_parenting_report

- 이름:
  - `유아·아동 기질 리포트`
- 설명:
  - 기질, 반응 패턴, 안정 조건, 양육 포인트를 중심으로 보는 부모 상담형

## 4. 실연동 심사 전 최소 체크리스트

- 메인 페이지 존재
- 상품 소개 페이지 존재
- 신청 페이지 존재
- 보고서 열람 페이지 존재
- 환불 정책 페이지 존재
- 개인정보처리방침 존재
- 이용약관 존재
- 문의 이메일 또는 연락처 표시
- 사업자 정보 표시 가능하면 추가

## 5. 구현 우선순위

1. 메인
2. 상품 소개
3. 차트 신청
4. 환불 정책
5. 개인정보처리방침
6. 이용약관
7. 보고서 열람

## 6. 디자인 방향

- 크림 배경
- 골드 브라운 포인트
- 딥 네이비 텍스트
- 점술 사이트보다 프리미엄 해설 리포트 서비스 느낌
- 과장된 신비주의보다 차분하고 신뢰감 있는 문체
